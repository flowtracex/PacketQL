# Contributing to PCAPQL

Thanks for helping improve PCAPQL.

## How to Contribute

1. Open an issue describing the bug/feature.
2. Fork the repo and create a branch.
3. Keep changes focused and include tests when possible.
4. Open a PR with:
   - what changed
   - why it changed
   - how it was tested

## Development Notes

- Frontend: `ndr-frontend/gui`
- API: `ndr-api/flowtracex_api`
- Enrich pipeline: `ndr-enrich`

## Code Style

- Prefer small, reviewable PRs.
- Avoid unrelated refactors in the same PR.
- Keep docs updated when behavior changes.

## Reporting Security Issues

Do not open public issues for sensitive vulnerabilities.
Create a private report channel before public disclosure.
