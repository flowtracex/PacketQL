What analysts see

  • Search logs with fast pivots
  • Run SQL over structured investigation data

Architecture Flow

The product pipeline is intentionally simple for analysts, while still using a strong backend processing flow.

Step 1PCAP Upload

Users start with a packet capture.

Step 2Zeek

Zeek parses traffic into protocol-specific logs.

Step 3Kafka

Streaming events are buffered through Kafka in KRaft mode.

Step 4Go Pipeline

Events are normalized and enriched in real time.

Step 5Parquet + DuckDB

Structured outputs become SQL-queryable investigation data.

Step 6UI

Analysts investigate with dashboard, search, and SQL.
